Ejemplo de Creacion y insercion de datos:

En SQL los tipos de datos son como las “etiquetas” que le ponemos a cada columna para indicarle a la base de datos qué tipo de información puede guardar y cómo debe manejarla. Estos son algunos:

  • Int/Integer
  • Bigint
  • Char
  • Varchar
  • DateTime
CREATE TABLE pedidos (
    id_pedido INT PRIMARY KEY IDENTITY(1,1),
    id_usuario INT NOT NULL,
    producto VARCHAR(100) NOT NULL,
    cantidad INT NOT NULL,
    fecha_pedido DATE DEFAULT GETDATE(),
    FOREIGN KEY (id_usuario) REFERENCES usuario(id) -- Asume que usuario tiene columna 'id'
);

INSERT INTO pedidos (id_usuario, producto, cantidad) 
VALUES (2, 'Laptop', 1),  -- Asume que usuario con id=1 existe
       (4, 'Teléfono', 2),
       (5, 'Tablet', 1);

C.R.U.D:

CRUD (Create, Read, Update, Delete) son las operaciones fundamentales para el manejo de cualquier base de datos. Permiten crear nuevos registros, leer o consultar información, actualizar datos existentes y eliminar registros. Con estas acciones se cubre la gestión básica y esencial de los datos. 

select * from pedidos;
update pedidos set producto = 'relojSmart' where id_pedido = 4; 
delete from pedidos where id_pedido=5;

INNER JOIN, LEFT JOIN y RIGHT JOIN:

Son tipos de combinaciones de tablas que permiten unir datos de dos o más tablas mediante una columna en común. El INNER JOIN devuelve solo los registros que tienen coincidencia en ambas tablas. El LEFT JOIN devuelve todos los registros de la tabla izquierda y solo las coincidencias de la tabla derecha. El RIGHT JOIN hace lo opuesto: devuelve todos los registros de la tabla derecha y solo las coincidencias de la izquierda. Estas uniones facilitan obtener información más completa y detallada en una consulta.

INNER JOIN

Ejemplo de INNER JOIN: tengo tabla usuario y pedidos

select u.nombre, u.email, p.producto, p.cantidad, p.fecha_pedido
	  from usuario u
	  inner join pedidos p on u.id = p.id_usuario;
